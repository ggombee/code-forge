# Techniques - Chain of Thought & Extended Thinking

## Chain of Thought (CoT)

단계별 사고 유도 → 정확성 향상.

### 3단계 수준

| Level | Method | When | Example |
|-------|--------|------|---------|
| **Basic** | "단계별로 생각하세요" | 간단한 작업 | 단순 계산 |
| **Guided** | 구체적 단계 제공 | 중간 복잡도 | 데이터 분석 |
| **Structured** | `<thinking>` 태그 | 복잡한 작업 | 아키텍처 설계 |

### Structured Pattern (권장)

```xml
<thinking>
1. 문제 분석
   - 현재 상태 파악
   - 요구사항 정리

2. 접근 방법
   - 가능한 옵션들
   - 장단점 비교

3. 단계별 추론
   - 선택한 방법
   - 구현 단계

4. 검증
   - 예상 결과
   - 잠재적 문제
</thinking>

<answer>
최종 답변
</answer>
```

### 사용 시점

```text
✅ 사용하면 좋음
- 복잡한 수학 문제
- 다단계 분석
- 의사결정 (여러 옵션 비교)
- 디버깅 (원인 추적)
- 아키텍처 설계

❌ 사용 안 함
- 단순 검색
- 단순 변환 (JSON → CSV)
- 지연 시간 중요 (실시간)
- 사실 확인
```

## Extended Thinking

긴 추론 시간으로 복잡한 문제 해결.

### Best Practices

```text
❌ 단계별 지시 (제한적)
"1. 변수 식별
 2. 방정식 세우기
 3. 풀이
 4. 검증"

✅ 일반적 지시 (자유로움)
"철저히 생각하세요.
 여러 접근법을 고려하고,
 각각의 장단점을 평가하세요."

⚠️ 최소 토큰: 1024
⚠️ 32K+ 예산: Batch Processing 권장
```

### 사용 시점

| Use Case | Example | 왜 Extended? |
|----------|---------|-------------|
| 복잡한 STEM | 4D 시각화, 고급 물리 문제 | 다단계 추상화 |
| 제약 최적화 | 다중 제약 여행 계획 | 조합 탐색 |
| 사고 프레임워크 | Porter's 5 Forces + 시나리오 분석 | 구조적 사고 |
| 창의적 문제해결 | 새로운 알고리즘 설계 | 탐색적 사고 |

---

## CoT vs Extended

| 측면 | CoT | Extended |
|------|-----|----------|
| **복잡도** | 중간 (3-5 단계) | 매우 높음 (10+ 단계) |
| **토큰** | 100-500 | 1024+ |
| **제어** | 구조화된 단계 | 자유로운 탐색 |
| **속도** | 빠름 | 느림 |
| **사용** | 일반적 문제 | 극도로 복잡한 문제 |

---

## 요약

```text
간단 → 직접 답변
중간 → CoT (<thinking>)
복잡 → Extended Thinking (1024+)
```
