# Scope Completeness (스코프 완전성 검증)

> "모든 X를 수정해" 요청에서 일부만 처리하고 완료 선언하는 것을 방지한다

---

## 핵심 규칙

### 보수적 스코핑 금지

| 금지 | 필수 |
|------|------|
| 일부만 수정하고 "완료" 선언 | 전체 대상을 먼저 열거 후 작업 |
| "나머지는 비슷하게 하면 됩니다" | 실제로 모두 수정 |
| 대표 케이스만 처리 | 모든 케이스 처리 |

---

## 전수 조사 프로토콜

### Step 1: 대상 전체 목록 확인

```typescript
// "모든 컴포넌트에서 any를 제거해" 요청 시

// ❌ 일부만 확인
Grep("any", "src/components/OrderCard.tsx")

// ✅ 전체 대상 열거
Grep(": any", "src/", type="ts")  // 전체 스캔
// → 결과: 15개 파일에서 23건 발견
```

### Step 2: 사용자 확인 (선택)

대상이 많을 경우 먼저 범위를 확인:

```markdown
총 23건의 `any` 사용을 발견했습니다:
- src/components/ (12건)
- src/hooks/ (5건)
- src/utils/ (6건)

모두 수정할까요, 특정 디렉토리만 할까요?
```

### Step 3: 전수 처리

확인된 모든 대상을 처리한다. 누락 금지.

### Step 4: 완료 검증

```typescript
// 처리 후 재스캔으로 누락 확인
Grep(": any", "src/", type="ts")
// → 결과: 0건 (완전히 처리됨)
```

## 자주 발생하는 불완전 스코프

| 패턴 | 위험 | 방지 |
|------|------|------|
| "레이아웃만 변경" | 스타일/로직 누락 | 관련 스타일+로직 함께 확인 |
| "타입만 수정" | import/사용처 누락 | 모든 참조처 확인 |
| "컴포넌트만 추가" | 라우팅/export 누락 | 연결점 모두 확인 |
| "API 연동만" | 에러 처리/로딩 누락 | UX 상태 모두 확인 |
