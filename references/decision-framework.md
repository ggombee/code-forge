# Decision Framework

2-3 옵션 제시 포맷 (테이블 구조), 비교 기준 (방법/장점/단점/영향도), 사용자 선택 UI.

---

## Bug Fix — 해결 옵션 제시 포맷

반드시 2-3가지 옵션 제시:

```markdown
## 버그 분석 결과

### 증상

{증상 설명}

### 원인

{근본 원인 분석}

### 해결 옵션

| #   | 방법    | 장점   | 단점   | 영향 범위 |
| --- | ------- | ------ | ------ | --------- |
| 1   | {방법1} | {장점} | {단점} | {파일 수} |
| 2   | {방법2} | {장점} | {단점} | {파일 수} |
| 3   | {방법3} | {장점} | {단점} | {파일 수} |

### 권장

{권장 옵션 번호} - {이유}

어떤 방법으로 수정할까요? [1/2/3]
```

```markdown
### [입장 B] 주장

**위험 완화:** {예상 단점 대응}
```

### 라운드 2: 반론

```markdown

**[입장 B] 주장의 약점:** {구체적 지적}
```

```markdown
**인정하는 부분:** {상대 주장 중 수용 가능한 부분}
```

## /my-tickets — AskUserQuestion 선택지 UI

반드시 `AskUserQuestion` 도구를 사용하여 선택지를 제시한다. 텍스트가 아닌 클릭 가능한 UI로.

```
question: "작업을 선택하세요"
options:
  - "QA-52372 그룹 (3건) — 자동이체 노출"
  - "QA-52339 — 0원 광고 즉시 취소"
  - "QA-52316 — 부속상품 advertisementType"
  - "QA-52141 — paymentMethodType 재요청"
  - "🔄 전체 처리 (우선순위 + 코드 영향 순서)"
```

각 옵션 텍스트에 **티켓번호를 반드시 포함**하여, 사용자가 선택하면 티켓번호를 파싱하여 `/start`로 연계.

## /done 완료 후 처리 방식 선택

```
모든 티켓 처리가 끝나면:

전체 처리 완료 ({N}건)

{처리 결과 테이블}

/done 실행할까요? (커밋 + PR + Jira 완료 처리)
  a. 전체 /done (모든 변경사항을 한 커밋 + 한 PR)
  b. 티켓별 /done (각 티켓마다 커밋 + PR 분리)
  c. 나중에 할게요
```
