## Phase 2: Analysis Primitives (1주)

> CMS 핸드오프에서 검증된 패턴. 4개 스킬에서 실제 중복 확인됨.

### 2-1. Primitive instruction 파일 5개 생성

위치: `instructions/analysis/`

| 파일 | 용도 | 중복 제거 대상 |
|------|------|---------------|
| `input-resolver.md` | 입력 유형 판별 (티켓/MD/텍스트) | /start 0단계 |
| `complexity-judge.md` | 복잡도 판단 LOW/MED/HIGH | /start, /bug-fix, /refactor, thinking-model (4곳) |
| `code-explorer.md` | 병렬 탐색 차원 템플릿 | /start, /bug-fix, /refactor (3곳) |
| `policy-detector.md` | 정책 키워드 감지 → 테스트 전략 | /done, /refactor (2곳) |
| `figma-analyzer.md` | Figma MCP/Curl/Skip 모드 | /start, /figma-to-code (2곳) |

`plan-generator.md`는 우선순위 낮음 (각 스킬 출력이 충분히 다르므로).

### 2-2. `/analyze` 스킬 생성

`/start`에서 워크플로우(Jira 상태 변경, Gate, 구현 확인)를 제거한 순수 분석 스킬:

```yaml
---
name: analyze
description: 요구사항을 분석하여 수정 계획을 출력한다. 워크플로우 없이 순수 분석만 수행.
category: analysis
---
```

5개 primitive를 @참조로 조합:
```
/analyze TICKET-123
  → [input-resolver] 입력 판별
  → [code-explorer] 병렬 탐색
  → [complexity-judge] 복잡도 평가
  → [policy-detector] 정책 감지
  → [figma-analyzer] Figma 분석 (링크 있으면)
  → 분석 결과만 출력하고 종료
```

### 2-3. 기존 스킬 리팩토링 (primitive @참조 전환)

동작은 동일하게 유지 (회귀 없음). 인라인 로직을 @참조로 교체:

| 스킬 | 참조할 primitive |
|------|-----------------|
| `/start` | input-resolver + figma-analyzer + code-explorer + complexity-judge |
| `/bug-fix` | complexity-judge + code-explorer |
| `/refactor` | code-explorer + complexity-judge + policy-detector |
| `/done` | policy-detector |

### 2-4. Dead reference 정리

현재 /start가 참조하는 존재하지 않는 파일 2개:
- `instructions/workflow-patterns/sequential-thinking.md` → 제거 또는 생성
- `instructions/validation/release-readiness-gate.md` → 제거 또는 생성

## Phase 3: 사용량 로깅 + /stats (1주)

### 3-1. usage.jsonl 로깅

위치: `~/.code-forge/usage.jsonl`

```json
{"skill":"bug-fix","ts":"2026-03-19T10:00:00Z","project":"a1b2c3","success":true}
```

- 프로젝트는 SHA-256 해시 (경로 미노출)
- 코드 내용 절대 미기록
- 로컬 only, 외부 전송 없음
- hooks에서 스킬 호출 감지하여 기록

### 3-2. `/stats` 스킬 생성

```yaml
---
name: stats
description: 스킬 사용량 통계를 터미널에 출력한다.
category: utility
---
```

usage.jsonl 파싱 → 터미널 출력:
- 최근 7일 스킬 사용 빈도 top 5
- 에이전트 호출 빈도
- 프로젝트별 활동량

### 3-3. /anvil-build 보안 스캔 추가

빌드 시점에 SKILL.md/agent.md grep 기반 위험 패턴 체크:
