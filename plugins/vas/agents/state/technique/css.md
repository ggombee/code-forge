## Must
- [Specificity] 선택자 우선순위를 의도적으로 관리한다
- [Responsive] 반응형 디자인을 모바일 퍼스트로 작성한다
- [TokenUsage] 디자인 토큰(변수)을 하드코딩 값보다 우선 사용한다

## Never
- [ImportantOveruse] !important를 레이아웃/테마 오버라이드 외에 사용하지 않는다

## Should
- [Naming] 일관된 네이밍 컨벤션(BEM, utility 등)을 따른다
